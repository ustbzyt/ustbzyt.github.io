Automation of High-Throughput LC-MS Intact Mass Analysis Workflow for Antibody Biologics

Organization: Bristol Myers Squibb (as per your Profile)

Date: Recent (e.g., 2023 - 2024, please fill in the specific time)

The Challenge:

High-throughput LC-MS intact mass analysis of antibody biologics (including bispecific antibodies, fusion proteins, etc.) generates a large volume of data. Manual operations for both instrument run sequence setup and subsequent data analysis preparation are time-consuming and prone to errors.

My Solution:

I developed a Python-based toolkit to significantly streamline this process:

  • Automated Sequence Generation:
    • The script utilizes plate map information provided by users (colleagues submitting samples for LCMS analysis).
    • It automatically generates a CSV file that can be directly imported into ThermoFisher Orbitrap mass spectrometer's Xcalibur software as an instrument run sequence. This eliminates tedious manual input of sample names, positions, instrument methods, file save paths, and injection volumes.
    • Concurrently, it generates a second CSV file specifically for the intact mass spectrometry data analysis software, Protein Metrics (Byos). After samples are analyzed on the mass spectrometer, raw data can be placed in a designated folder, and this CSV can be imported into Byos to initiate batch data analysis with minimal additional operations.
  • User-Friendly Design:
    • The script is equipped with a Graphical User Interface (GUI) for ease of use by colleagues without a programming background.
    • It integrates automated features, such as automatic installation of required Python packages if they are missing on a colleague's computer.
    • It offers various customizable settings, such as mass spectrometry data naming conventions and default data storage locations, to accommodate different user and project needs.

Impact:

  • Significantly reduced manual effort and time required for LC-MS run setup and data analysis initialization.
  • Minimized human errors during sample information entry.
  • Increased the overall efficiency and throughput of the intact mass analysis workflow.
  • Improved user experience for instrument operators and data analysts.

Technologies Used: Python, GUI Development (e.g., Tkinter, PyQt), CSV file processing.

Advanced Automation and Visualization of Protein Metrics Software Analysis Reports

Organization: Bristol Myers Squibb (as per your Profile)

Date: Recent (e.g., 2023 - 2024, please fill in the specific time)

The Challenge:

After processing data with Protein Metrics software, writing comprehensive and easily understandable comments on experimental results for non-mass spectrometry specialists, and visualizing 96-well plate results for trend analysis, are both time-consuming and critical steps.

My Solution:

I developed a powerful program with two core functionalities:

  • AI-Driven Automated Comment Generation:
    • The program integrates report data exported from Protein Metrics with metadata from the plate map.
    • This consolidated information is sent to a cloud-based Large Language Model (LLM), currently OpenAI's GPT-4.1.
    • Through careful prompt engineering to constrain the model's output, the system automatically generates expert-level, yet concise and easy-to-understand, comments on experimental results, greatly facilitating communication with non-specialist team members.
  • Interactive 96-Well Plate Heatmap Visualization:
    • The program processes report data exported from Protein Metrics software.
    • Based on this data, it automatically generates an interactive HTML-formatted 96-well plate heatmap.
    • This visualization clearly displays key metrics for each well, such as data quality and impurity levels, helping users quickly identify trends, assess sample quality, and make informed decisions.

Impact:

  • Drastically reduced the time and effort required for manually writing report comments.
  • Improved the clarity and consistency of reports for a broader audience.
  • Enabled rapid identification of data trends and outliers through intuitive visualization.
  • Accelerated data-driven decision-making processes for project teams.

Technologies Used: Python, OpenAI API (GPT-4.1), Prompt Engineering, HTML, CSS, JavaScript (potentially for interactive visualization), Data Parsing.
